使用Vscode写springboot项目

Apr 06, 2022

并不推荐大家使用Vscode写SpringBoot项目结构复杂，Vscode的插件管理始终不如IDEA的软件管理来的方便，如果你想多一种选择或者想尝试手动管理的“快感”的话就请看下去吧

确定依赖及环境变量配置正确

接下来将会使用Scoop包管理器进行自动安装，关于Scoop可以查看博客文章或者查看其官网或者百度

Maven 和 MAVEN_HOME

安装Maven

1

scoop install maven

添加MAVEN环境变量

在环境变量中新建一个变量MAVEN_HOME

路径选择C:\Users\用户名\scoop\apps\maven\current

Java 和 JAVA_HOME

安装Java

1
2
3

scoop bucket add java               # 添加java的bucket
scoop install java/openjdk17        # java 17
scoop install java/openjdk-redhat   # java 8

添加JAVA环境变量

在环境变量中新建一个变量JAVA_HOME

路径选择C:\Users\MacKenia\scoop\apps\openjdk17\current

安装Vscode插件

Vscode本身是一个文本编辑器，但是在庞大的插件市场的支撑下，让它无所不能

需要安装的插件有:

必装:

  1. Java
    1. Extension Pack for Java
    2. Debugger for Java
    3. Project Manager for Java
    4. Test Runner for Java
    5. Language Support for Java
  2. Spring-boot
    1. Spring Boot Dashboard
    2. Spring Boot Tools
    3. Spring Initializr Java Support

花里胡哨的插件:

  1. Colonize
  2. ErrorLens
  3. GitHub Copilot

初始化项目

打开Vscode

按下Ctrl+Shift+p打开命令面板

输入Spring Initializr:Create a Maven Project...，并回车

选择任意Maven版本

语言选择Java

输入Group ID

输入项目名称

选择项目的打包输出格式，建议使用Jar

选择Java版本

选择依赖，可以直接搜索，添加spring web，回车选择

依赖添加完成后回车，选择项目创建的路径，即可完成项目的创建

启动Spring Boot项目

此时可以直接按下F5，使用Vscode提供的默认调试方法来启动项目，但是不推荐该方法。

推荐使用Spring Boot Dashboard中提供的调试方法

更换Maven仓库

在使用Scoop安装的Maven的情况下

打开C:\Users\MacKenia\scoop\apps\maven\current\conf

即可找到Maven的配置文件，在mirrors块中添加国内源mirror，并且注释原mirror即可

阿里源:

1
2
3
4
5
6

<mirror>
    <id>nexus-aliyun</id>
    <mirrorOf>central</mirrorOf>
    <name>Nexus aliyun</name>
    <url>http://maven.aliyun.com/nexus/content/groups/public</url>
</mirror>

添加依赖

在左侧的资源管理器下方，有一个叫做Maven的标签，点开即可看到Dependencies栏

点击该栏后，会出现一个加号，以及一个刷新键

若在pom文件中更新了依赖，则点击刷新键重新拉取云端依赖

用于便捷添加依赖，提供云端仓库搜索功能
